⚠ List View Design Decisions

  • Grouping: Shown grouped by column (Inbox/Backlog/In Work/Done). Could also group by priority, due date, or none (flat). Should grouping be toggleable?
  • Sorting: Within each group, tasks are sorted by priority (high first) then due date. Should sorting be user-configurable (click column headers)?
  • Column indicator: Each row shows a "Backlog" / "In Work" pill. In list view this replaces the spatial grouping of Kanban. Is it redundant with the group headers?
  • Done visibility: In Kanban, Done shows "last 10". In list view, should Done show all? Or also cap at 10? Current mockup shows 3 for brevity.
  • Row click: Currently toggles checkbox. Should it open a detail view instead? (Checkbox click would be a separate target.)
  • Inline editing: Should clicking the title allow inline rename? Or require opening a detail panel?
  • Drag reorder: Not shown here. Should list rows be draggable to change column assignment (like dragging to a different group)?
  • Note link column: Shows note title for linked tasks. Takes horizontal space. Worth keeping, or hide behind hover/click?
