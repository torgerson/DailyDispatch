⚠ Kanban Design Decisions

  • Priority indicator: Showing both a left-edge stripe AND a dot. Stripe is more visible on a board; dot is more compact. Which to keep?
  • Card click behavior: Click opens a detail panel/modal? Or inline expand? Currently just the checkbox is interactive.
  • Column header color: Each column has a colored bottom border (accent for regular, blue for In Work, green for Done). Should users pick column colors?
  • Quick-add: "+ Add task" at the bottom of each column — does it add to THAT column, or always to Inbox? Spec says Inbox, but per-column add feels more natural on a Kanban board.
  • Note-linked indicator: Small "📄 Q1 Planning" link on the card. Click opens the parent note. Is the label (note title) needed, or just an icon?
  • Done column count: Spec says "last 10". Showing 3 here for space. The "Showing last 3 of 6 completed" footer communicates the cap. Is 10 the right number?
  • Drag-to-reorder cards: Cards are draggable="true" in the HTML. Full drag-drop not wired (this is a mockup). Cross-column drag would move cards; within-column drag reorders.
  • Empty column: What does an empty column look like? Probably just the header + the "+ Add task" button, no special empty state needed?
