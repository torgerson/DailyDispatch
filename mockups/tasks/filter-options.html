Option A — All Chips

Every filter is a pill chip. No dropdowns at all. Priority chips toggle on/off individually. Date filters are also chips — only one active at a time (radio behavior). Clean, flat, everything visible.

Option B — Segmented Controls

Option C — Minimal Text Filters

No borders, no pills — just text labels that light up on click. Counts shown inline. Feels like a toolbar, not a form. Lightest visual weight. Priority dots + text, date as plain text toggles.

Design Notes

  • Option A (Chips): Most visual. Every state is obvious. May feel busy with 10+ chips on a narrow screen.
  • Option B (Segments): Most compact. Groups related filters logically. iOS-native feel. May feel "buttony".
  • Option C (Minimal): Lightest weight. Feels editorial. Counts inline are useful. May be too subtle for discoverability.
  • All three remove the native <select> dropdown. The date range becomes visible toggle buttons instead of a hidden menu.
  • Priority is always multi-select (toggle each independently). Date is single-select (radio — only one active).
  • Search input is consistent across all three: rounded, compact, left-aligned.
