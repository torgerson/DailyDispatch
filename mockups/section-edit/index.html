Tradeoffs

A · Guided Setup

Strongest first-run clarity — impossible to miss the name/filter fields
Chips teach query syntax by example, clickable to insert
Visually distinguishes new vs. saved
Two visual modes = more code / CSS
Experts doing bulk edits have to enter edit mode each time

B · Always-Visible

Zero hidden state, everything is already on screen
Smallest code change from today's design
Edit & preview are unified — no mode switching
Takes more vertical space per section
Empty-state still looks "ready" — slightly less push to fill it in

C · Card + Modal

Cleanest sidebar when you have many sections
Modal gives room for help text, examples, delete
Separates "use" from "configure" cleanly
Modals are friction for quick tweaks
First-run discoverability depends on finding the gear icon
