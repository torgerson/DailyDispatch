DailyDispatch wants to be the work journal you didn't know you needed

A single HTML file that combines note-taking, task management, and a daily feed. It's ambitious, opinionated, and surprisingly deep — but it's got some rough edges to sand down.

By Mock Reviewer / Apr 12, 2026, 10:00 AM EDT

DailyDispatch

DailyDispatch running in Chrome with the Warm Notion theme. The sidebar, feed timeline, and task widgets are all rendered from a single HTML file.

There's a category of productivity tool that's hard to explain to normal people. It's not a to-do list. It's not a note app. It's not a journal. It's... all of them, mashed together, for the kind of person who thinks in markdown and gets annoyed when software phones home. DailyDispatch is one of those tools, and it's one of the more interesting entries I've seen in a while.

The pitch: a local-first work journal where notes and tasks live together, stored entirely in your browser's IndexedDB, delivered as a single HTML file. No account. No cloud. No install. Just open it in Chrome and start typing.

I've been using it for about a week, and I have thoughts.

What it gets right

The daily feed is genuinely great

The centerpiece of DailyDispatch is a chronological feed of everything you type during the day. Quick entries, meeting notes, ideas, deployment logs — it all lands in a timeline grouped by day and hour, with little amber dots marking notes versus quick entries. It feels like a work-specific Twitter feed, except private and searchable.

The "command card" at the top of the Today view is a smart design. The capture input and your due tasks sit in one bordered card, so you always know what to do and where to type. The cursor auto-focuses when you switch to Today. Small detail, good instinct.

The feed makes you feel like your day is being recorded, not managed. That's a surprisingly different feeling from a to-do list.

Tasks inside notes is the killer feature

Most note apps treat tasks as decoration. You type a checkbox, and that's it — there's no board, no priority, no due date. It's just text that looks like a checkbox. DailyDispatch actually connects them. Type /task in a note and you get a linked task that shows up on a Kanban board, has a priority level, a due date, a description field, and a comment log. Complete it on the board and it updates in the note. That bidirectional link is the kind of thing that sounds obvious but almost nobody does well.

The Smart List feature extends this further — type /smartlist in any note and you get a live search widget embedded inline, showing matching entries. Want a "Project X" note that always shows everything tagged #projectx? Done. It turns notes into dashboards.

Search is a first-class citizen

The search syntax is surprisingly powerful: tags, mentions, boolean OR, phrase matching, negation, and a line: scope operator. Each search opens as a persistent tab you can keep alongside your notes. The sidebar sections are essentially pinned searches that update live. For a single-file app, this is impressive.

The privacy story is real

I inspected the network tab. Zero requests. Not even a favicon fetch from a CDN. Everything — the CodeMirror editor, the Kanban board, 15 themes, the entire help system — is bundled into one HTML file. Your data never leaves IndexedDB. For anyone who's paranoid about cloud note services reading their meeting notes, this is exactly what they want to hear.

It's the most feature-dense single HTML file I've ever opened. That's both impressive and slightly terrifying.

Where it gets confusing

The learning curve is steeper than it needs to be

DailyDispatch has a lot of surface area. Slash commands, Smart Lists, sections, tags, mentions, cross-links, Kanban columns, task widgets, status badges, templates, a floating note window — it's all there. But discoverability is weak. You have to know to type / to get the command menu. You have to know that clicking a tag in the sidebar filters the feed. The help section is thorough, but it's buried in a sidebar section that most users will collapse and forget about.

The onboarding homepage is polished but doesn't actually teach you anything interactively. There's no guided tour, no "try this now" prompts. You read the marketing copy, click "Go to Today," and you're on your own.

The sidebar is getting crowded

Calendar, Tags, Sections header, section panels — there's a lot of stuff stacked vertically in a 240px column. When you have a few sections expanded, the tags get pushed up under the calendar and the sections scroll independently. It works, but it feels like four different UI concepts fighting for the same narrow strip. I'd love to see a collapsible sidebar or a way to toggle between "navigation mode" and "dashboard mode."

The Kanban board is functional but basic

Drag-and-drop works. Column management works. The amber drop indicators are a nice touch. But compared to dedicated tools like Linear or Trello, it feels thin. There's no swimlanes, no card labels beyond priority, no subtasks, no due date calendar view. For light task tracking alongside notes, it's fine. For anyone doing serious project management, they'll outgrow it quickly.

Some UI elements feel unfinished

The column selector in the task modal is still a native <select> dropdown, which looks out of place next to the priority pills. Some modals (like the generic confirm/prompt dialogs) are clean, but others (the section edit card) feel like they belong to an earlier design era. The header band at the top is just a colored strip with no content — it works as a separator from browser tabs, but it feels like wasted space.

What I'd want to see next

Mobile. Something, anything.

This is desktop-first by design, and the CLAUDE.md file explicitly says mobile is a non-goal. But in 2026, a tool that can't capture a quick thought from your phone is leaving value on the table. Even a basic responsive view that lets you add entries and check off tasks would be a huge win.

Better onboarding

An interactive walkthrough that says "type something here and press Cmd+Enter" and then "now type /task and see what happens" would go a long way. The help content is excellent — 14 detailed pages — but nobody reads help proactively.

Collaboration, even just a little

I get that this is a personal journal. But the @mentions feature suggests awareness of other people. Being able to share a single note (read-only export, maybe as a markdown file or HTML snippet) would make this more useful in team contexts without compromising the local-first story.

File system sync

The File System Access API exists. Let users pick a folder and auto-save their data as JSON. That would unlock Dropbox/iCloud/Git sync without building any cloud infrastructure. Right now, the only backup option is manual JSON export, which most people will forget to do.

The Verdict

DailyDispatch is an opinionated, privacy-first work journal that gets the core loop right: capture a thought, tag it, find it later. The task integration is genuinely novel — most note apps don't even try — and the Smart List feature turns notes into living dashboards. For engineers, PMs, and anyone who lives in a browser and thinks in markdown, it's worth trying.

But it's not for everyone. The learning curve is real, the sidebar is getting dense, and the lack of mobile or sync means it's a single-browser, single-device tool. If you're okay with that trade-off — and you value privacy enough to accept it — there's nothing else quite like it.

7.5 / 10 — An ambitious, deeply personal tool that nails the intersection of notes and tasks. Needs polish, onboarding, and a mobile story to break out of the power-user niche.

Product Action Items (for the DailyDispatch team)

Fix now (polish & consistency)

  • Column selector in task modal — replace the native <select> dropdown with styled pills or a custom dropdown matching the priority pills
  • Section edit cards — restyle the setup/edit UI to match the modal design language (rounded corners, consistent spacing, accent colors)
  • Header band — either put something useful in it (breadcrumb, current note title) or reduce it to a thin 4px accent line
  • Feed entry previews — the token humanization is a good start, but "[table]" and "[list: #query]" still feel technical. Consider small inline icons instead
  • Empty states — the Kanban board with empty columns just shows blank space. Add helpful prompts ("Drag tasks here" or "Click + to add")

Build next (high-impact features)

  • Interactive onboarding — a 5-step guided tour that walks new users through capture, /task, tagging, and search. Show, don't tell
  • File System Access API sync — let users pick a folder for auto-save. Unlocks iCloud/Dropbox/Git sync with zero infrastructure
  • Markdown export per note — export a single note as .md file for sharing. Enables lightweight collaboration without cloud
  • Task subtasks — a task with child tasks would make the Kanban board actually useful for project breakdown
  • Quick-open command palette — Cmd+Shift+O should be a fuzzy-search palette (like VS Code's Cmd+P) that searches notes, tasks, tags, and commands
  • Recurring tasks — "every Monday: review PRs" is a common pattern that currently requires manual recreation

Consider for roadmap

  • Responsive/mobile view — even a basic capture + today view that works on a phone would expand the audience significantly
  • Sidebar modes — toggle between compact (just nav) and expanded (nav + calendar + tags + sections) to reduce visual density
  • Card labels/colors on Kanban — tags should show on cards, and custom color labels would help visual scanning
  • Note templates with variables — {{date}}, {{time}} already work, but add {{cursor}} to place the cursor after insertion
  • Calendar integration — show tasks with due dates on the calendar grid, not just entry dots
  • Undo for destructive actions — "Note deleted. Undo?" toast instead of permanent deletion

Consider cutting or simplifying

  • Events & Activity Log — most users don't care that they renamed a section. The search event "reopen" feature is useful; the rest is noise in the feed
  • Floating note window — great concept, but popup windows are unreliable across browsers and OS configurations. Consider an in-app side panel instead
  • Questions system — still referenced in the data model but not exposed in the UI. Either ship it or remove the dead code
  • Contexts — same as Questions. Referenced in the schema, never surfaced. Clean it up
  • Section resize handles — dragging to resize section panels is a power feature that adds complexity. Consider fixed sizes (compact/expanded toggle) instead
